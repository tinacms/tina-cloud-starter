TinaCMS wurde entwickelt, um Content-Ersteller mit einer intuitiven und flexiblen Bearbeitungserfahrung zu stärken, und unsere Markdown-Unterstützung ist keine Ausnahme. Ob du einen einfachen Blog oder eine komplexe Dokumentationsseite verwaltest, TinaCMS integriert Markdown nahtlos und kombiniert die Kraft strukturierter Inhalte mit der Einfachheit der Echtzeit-Bearbeitung. Von Überschriften und Links bis hin zu Bildern und benutzerdefinierter Syntax, TinaCMS sorgt dafür, dass deine Markdown-Dateien einfach zu verwalten und anzupassen sind, während du die vollständige Kontrolle über deine Inhalte behältst.

## Flussdiagramme

Verwende ein Flussdiagramm, um Prozesse, Entscheidungsbäume oder Arbeitsabläufe zu visualisieren. Es ist ideal, um komplexe Systeme zu vereinfachen oder Logik visuell zu erklären.

```mermaid
flowchart TD
    A(Napoleon's Aspirationen) --> B{Benötigte Fähigkeiten?}
    B -->|"Nunchaku-Fähigkeiten"| C[Nunchaku-Training]
    B -->|Computer-Hacking-Fähigkeiten| D[Hacking-Praxis]
    B -->|Bogenjagd-Fähigkeiten| E[Bogenschützen-Club]
    C --> F[Werde ein Kampfkunst-Meister]
    D --> G[Erstelle das ultimative Chat-Programm]
    E --> H[Gewinne eine Jagd-Trophäe]
    F --> I{Mädchen beeindrucken?}
    G --> I
    H --> I
    I -->|Ja| J[Date mit Deb]
    I -->|Nein| K[Zeichne Liger]
    K --> L[Werde trotzdem großartig]
```

## Sequenzdiagramm

Verwende ein Sequenzdiagramm, um Interaktionen zwischen Komponenten oder Systemen über die Zeit zu veranschaulichen, besonders für APIs, Arbeitsabläufe oder Systeminteraktionen.

```mermaid
sequenceDiagram
    participant Frodo
    participant Sam
    participant Ring
    Frodo->>Sam: "Wir müssen ihn zerstören."
    Sam->>Ring: Trägt die Last
    Ring-->>Sam: "Es ist zu schwer!"
    Sam-->>Frodo: "Ich kann ihn nicht tragen, aber ich kann dich tragen!"
    Frodo-->>Ring: Wirft ihn in den Schicksalsberg
```

## Klassendiagramm

Verwende ein Klassendiagramm für objektorientiertes Systemdesign, das die Struktur von Klassen, Attributen und ihren Beziehungen zeigt.

```mermaid
classDiagram
    class Jedi {
        +String name
        +String lightsaberColor
        +useForce()
    }
    class Sith {
        +String name
        +String lightsaberColor
        +useDarkSide()
    }
    Jedi <|-- Sith
```

## Zustandsdiagramm

Verwende ein Zustandsdiagramm, um verschiedene Zustände eines Systems oder Objekts und Übergänge zwischen diesen Zuständen darzustellen.

```mermaid
stateDiagram-v2
    [*] --> TheShire
    TheShire --> Rivendell: Beginne die Reise
    Rivendell --> Moria: Durchquere die Minen
    Moria --> Gondor: Kämpfe gegen die Mächte des Bösen
    Gondor --> Mordor: Nähere dich dem Schicksalsberg
    Mordor --> [*]: Mission abgeschlossen
```

## Entity-Relationship-Diagramm

Verwende ein ER-Diagramm, um Datenbankstrukturen zu modellieren und Entitäten, Attribute und Beziehungen zu zeigen.

```mermaid
erDiagram
    AVENGER ||--o{ MISSION : participates
    MISSION ||--|{ ENEMY : targets
    AVENGER {
        string name
        string superPower
    }
    MISSION {
        int id
        string location
    }
    ENEMY {
        string name
        string weakness
    }
```

## Requirements-Diagramm

Verwende ein Requirements-Diagramm, um Projektanforderungen und ihre Beziehungen zu verfolgen.

```mermaid
    requirementDiagram

    requirement test_req {
    id: 1
    text: der test text.
    risk: high
    verifymethod: test
    }

    element test_entity {
    type: simulation
    }

    test_entity - satisfies -> test_req
```

## C4-Diagramm

Verwende ein C4-Diagramm für Software-Architektur, das Systemkontexte, Container, Komponenten und Code zeigt.

```mermaid
    C4Context
      title System Context Diagramm für Internet Banking System
      Enterprise_Boundary(b0, "BankBoundary0") {
        Person(customerA, "Bank-Kunde A", "Ein Kunde der Bank mit persönlichen Bankkonten.")
        Person(customerB, "Bank-Kunde B")
        Person_Ext(customerC, "Bank-Kunde C", "desc")

        Person(customerD, "Bank-Kunde D", "Ein Kunde der Bank, <br/> mit persönlichen Bankkonten.")

        System(SystemAA, "Internet Banking System", "Ermöglicht es Kunden, Informationen über ihre Bankkonten einzusehen und Zahlungen zu tätigen.")

        Enterprise_Boundary(b1, "BankBoundary") {

          SystemDb_Ext(SystemE, "Mainframe Banking System", "Speichert alle wichtigen Bankinformationen über Kunden, Konten, Transaktionen, etc.")

          System_Boundary(b2, "BankBoundary2") {
            System(SystemA, "Banking System A")
            System(SystemB, "Banking System B", "Ein System der Bank mit persönlichen Bankkonten. nächste Zeile.")
          }

          System_Ext(SystemC, "E-Mail-System", "Das interne Microsoft Exchange E-Mail-System.")
          SystemDb(SystemD, "Banking System D Datenbank", "Ein System der Bank mit persönlichen Bankkonten.")

          Boundary(b3, "BankBoundary3", "boundary") {
            SystemQueue(SystemF, "Banking System F Queue", "Ein System der Bank.")
            SystemQueue_Ext(SystemG, "Banking System G Queue", "Ein System der Bank mit persönlichen Bankkonten.")
          }
        }
      }

      BiRel(customerA, SystemAA, "Verwendet")
      BiRel(SystemAA, SystemE, "Verwendet")
      Rel(SystemAA, SystemC, "Sendet E-Mails", "SMTP")
      Rel(SystemC, customerA, "Sendet E-Mails an")

      UpdateElementStyle(customerA, $fontColor="red", $bgColor="grey", $borderColor="red")
      UpdateRelStyle(customerA, SystemAA, $textColor="blue", $lineColor="blue", $offsetX="5")
      UpdateRelStyle(SystemAA, SystemE, $textColor="blue", $lineColor="blue", $offsetY="-10")
      UpdateRelStyle(SystemAA, SystemC, $textColor="blue", $lineColor="blue", $offsetY="-40", $offsetX="-50")
      UpdateRelStyle(SystemC, customerA, $textColor="red", $lineColor="red", $offsetX="-50", $offsetY="20")

      UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")
```
