## Markdown, Aber Besser

Hallo, das aktuelle Datum ist <DateTime format="local" />. Markdown ist großartig zum Schreiben von Inhalten, aber was ist, wenn Sie etwas Interaktiveres benötigen? Was ist, wenn Sie dynamische Daten abrufen, formatierte Callouts anzeigen oder Benutzereingaben sammeln möchten? Hier kommen Komponenten ins Spiel.

---

```graphql
query MyQuery($relativePath: String!) {
  page(relativePath: $relativePath) {
    title
  }
}
```

> Mit Komponenten ist Ihr Markdown-Inhalt nicht nur statischer Text—er wird zu einer dynamischen Erfahrung. Sie können Daten abrufen, Inhalte schön formatieren und sogar interaktive Elemente direkt in Ihre Beiträge einbauen.

## Warum Komponenten in Markdown verwenden?

- **Inhalt dynamisch halten** - Echtzeitdaten statt statischen Text abrufen.
- **Lesbarkeit verbessern** - Strukturierte Callouts, Buttons und Formulare verwenden.
- **Benutzerinteraktion erweitern** - Benutzer können direkt im Inhalt handeln.

## Weitere Möglichkeiten, Komponenten zu verwenden

1. **GraphQL-Abfragen einbetten** - Inhalte dynamisch abrufen und anzeigen.
2. **Gestaltete Callouts verwenden** - Wichtige Punkte mit visuell unterschiedlichen Elementen hervorheben.
3. **Interaktive Formulare erstellen** - Benutzer können sich anmelden, Daten übermitteln oder mit Ihren Inhalten interagieren.
